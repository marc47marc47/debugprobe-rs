debugprobe-rs — 測試接線圖 (TEST-plan)

Pico A = 探針 (Layer 1，跑 debugprobe-rs，USB 接主機) ｜ Pico B = 目標 (Layer 2)
兩板皆可接 SSD1306 0.96" OLED 獨立 debug；SWD 與 OLED 並存不衝突。
SWD / Reset UART 橋接 I²C OLED 3V3 電源 GND USB LED

連線對照表（實體腳位）

OLED 接線（兩板相同）

SCLGP7 (I²C1 SCL)pin 10

測試步驟與驗證指令

前提：Pico A 跑 debugprobe-rs（USB 接主機，序號 E6605838834DA330）； Pico A 燒錄需 BOOTSEL；Pico B 可由 Pico A 經 SWD 用 probe-rs 燒錄。
#測試指令 / 動作預期
1USB 列舉probe-rs listCMSIS-DAP v2 + CDC(COMx) ✓
2SWD 偵錯probe-rs info --chip RP2040 --protocol swd讀到目標 DP/ROM，與 C 一致 ✓
3SWD 燒錄probe-rs download ... uarthello燒入目標成功 ✓
4UART→USB目標跑 uarthello，讀 COMx收到 hello from target #N ✓
5USB→UART目標跑 uartmon，寫 COMxB 的 OLED 顯示 rx bytes: N + echo 回讀 ✓

產生測試韌體與燒錄

cargo build --release --no-default-features --features rp2040 --bin uarthello
cargo build --release --no-default-features --features rp2040 --bin uartmon
probe-rs download --chip RP2040 --probe "2e8a:000c-0:<serial>" --protocol swd --speed 1000 \
  target/thumbv6m-none-eabi/release/uartmon
probe-rs reset --chip RP2040 --probe "2e8a:000c-0:<serial>" --protocol swd
