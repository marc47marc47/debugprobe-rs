接線計畫 + 測試紀錄：Nucleo STM32F446RE 作為 layer 2 目標

✅ 已實機驗證 2026-06-16 用 debugprobe-rs 探針把 Nucleo-F446RE（STM32F446RET6, Cortex-M4F）當 layer 2：SWD 燒錄/重置 + USB-to-UART 橋接 + OLED + LED。純文字版見 TEST-stm32f446re.md。

關鍵 先移除 CN2 兩顆 "ST-LINK" 跳線——Nucleo 有板載 ST-LINK， 不移除會與外部探針在 SWD 上衝突（徵狀：DP 讀得到、AHB 失敗）。JP6(IDD) 保持插上、由 Nucleo 自身 CN1 USB 供電。

跳線插法

跳線設定原因
CN2（兩顆 "ST-LINK"）移除兩顆斷開板載 ST-LINK 與 onboard 目標的 SWD，讓外部探針獨佔
JP6（IDD）保持插上目標 MCU 供電路徑
電源選擇（U5V/E5V）保持預設 U5V由 Nucleo 自身 CN1 USB 供電

兩個 Nucleo 專屬眉角（實測）：
  1. 本顆 F446 帶 RDP Level 1（Nucleo 少見但確有）→ 跟 Black Pill 一樣先 stm32f2x unlock 0（mass erase）+ 拔插 CN1 USB 重新上電。
  2. NRST 與板載 ST-LINK 共線 → 任何走 reset 的燒錄（probe-rs download、OpenOCD program）都卡在「halt 後 reset timeout」。 但讀 DHCSR 發現 純 halt 其實成功（0x00030003，S_HALT=1）→ 繞道：halt → flash write_image → SYSRESETREQ run。

可靠燒錄流程（本板實際使用）

cd stm32f446-target && cargo build --release

# 1. 清 RDP（connect_assert_srst；印 "unlocked." 後拔插 CN1 USB 重新上電）
openocd -f interface/cmsis-dap.cfg -c "transport select swd" -c "adapter speed 1000" \
  -f target/stm32f4x.cfg -c "reset_config srst_only srst_nogate connect_assert_srst" \
  -c "init" -c "catch { reset halt }" -c "catch { halt }" -c "stm32f2x unlock 0" -c "reset run" -c "shutdown"

# 2. 燒錄：halt → flash write_image（避開會卡的 reset-halt）→ SYSRESETREQ run
openocd -f interface/cmsis-dap.cfg -c "transport select swd" -c "adapter speed 1000" \
  -f target/stm32f4x.cfg -c "reset_config none" -c "cortex_m reset_config sysresetreq" \
  -c "init" -c "catch { halt }" \
  -c "flash write_image erase stm32f446-target/target/thumbv7em-none-eabihf/release/stm32f446-target" \
  -c "catch { reset run }" -c "shutdown"

若你的 Nucleo NRST 無共線干擾，probe-rs download --chip STM32F446RETx 應可直接用。 判準：LD2(PA5) 閃、OLED 顯示 f446 layer2/tx{n} rx{n}、CDC COM @115200 收到 hello from f446 #n 且打字被 echo 回。
