接線計畫：Anycubic Vyper / TriGorilla+ v0.0.6 → layer 2

接線計畫 + 焊點定位指南（尚未實機） 用 debugprobe-rs 探針對 Anycubic Vyper 主板（GD32F103RET6, Cortex-M3）做 SWD layer 2。 純文字版見 TEST-trigorilla-vyper.md。

⚠️ 這是實體 3D 印表機主板,不是開發板。 燒錄會覆蓋 Anycubic 原廠韌體;若 RDP 開啟,清除會 mass erase 連 SD bootloader 一起清掉（之後 SD 卡更新失效,只能 SWD 刷回完整韌體）。 測試時請關 24V 電源、只用 USB 供電（或拔掉加熱器/馬達），避免誤觸加熱/馬達腳。動手前先備好可恢復的完整韌體。

如何定位 SWD 焊點（公開資料未記載）

TriGorilla+ v0.0.6 的 SWD 焊點位置 Anycubic 官方與社群皆未公開（Vyper 平常走 SD 卡 bootloader,少用 SWD）。 PA13/PA14 是 GD32F103 固定的 SWD 腳,需自己在板上定位。
  1. 找測試焊點群：GD32 旁/SD 卡座/CH340 附近常有一排沒接元件的小焊點或 via。
  2. 萬用表連通量測（最可靠）：斷電下,從 GD32 的 PA13≈pin46、PA14≈pin49（LQFP64,以 datasheet 為準）逐點測附近焊點,嗶聲即對應訊號。
  3. 直接焊飛線到晶片腳（最後手段）。

找到後拍照標記;先量 3V3 焊點確認 ~3.3V、確認 GND。

後續流程（實機階段,待另行授權）

# 1. 非破壞性連線（確認認得 GD32、檢查 RDP）
probe-rs info --chip STM32F103RETx --probe 2e8a:000c-0:E6605838834DA330 --protocol swd --speed 1000
# 2. 若 RDP=0 → 先備份原廠 flash（openocd dump_image）
# 3. 若要燒（RDP=1 需 openocd stm32f1x unlock 0，會 mass erase）→ 再燒測試韌體
#    UART 橋接測試建議用 USART1 PA2/PA3（WiFi 排針）;勿用接 LCD 的 PB10/PB11
